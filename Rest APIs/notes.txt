REST API
Representational State Transfer
-> Resource Manipulation via Representation
-> Resources must be identifiable via URIs
-> Client Sever De-coupling
-> Statelessness: no session between requests
HTTP Methods (CRUD)
GET - Retrieve a resource
POST - Create a new resource with given content
PUT/PATCH - Update an existing resource or create it if not exists
DELETE - Delete the resource 
Self Explanable Messages
Status Codes
200 OK, 404 Not Found, 400 Bad Request, 500 Server Error
